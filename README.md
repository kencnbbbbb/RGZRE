# Encryption Algorithm RGR

Учебное консольное приложение на C++ для демонстрации шифрования и дешифрования.

Проект сделан как модульная система с динамически подключаемыми библиотеками:

- основная программа: `encryption_rgr`;
- загрузчик библиотек: `PluginLoader.h`, `PluginLoader.cpp`;
- общий API плагинов: `PluginAPI.h`;
- динамические модули алгоритмов в папке `plugins/`.

## Алгоритмы

1. Шифр Рабина
2. Шифр Месси - Омуры
3. ChaCha20
4. RSA
5. Шифр Шамира
6. А1Я33

Каждый алгоритм хранится в отдельной паре файлов `.h` и `.cpp`, а при сборке превращается в отдельную динамическую библиотеку:

- `plugins/librabin.dylib` или `plugins/librabin.so`
- `plugins/libmassey_omura.dylib` или `plugins/libmassey_omura.so`
- `plugins/libchacha20.dylib` или `plugins/libchacha20.so`
- `plugins/librsa.dylib` или `plugins/librsa.so`
- `plugins/libshamir.dylib` или `plugins/libshamir.so`
- `plugins/liba1ya33.dylib` или `plugins/liba1ya33.so`

На macOS создаются `.dylib`, на Linux - `.so`.

## Сборка

```bash
make
```

Запуск:

```bash
./encryption_rgr
```

Для выхода из главного меню и возврата из меню выбранного шифра используйте `Q` или `q`.

Пересборка с нуля:

```bash
make clean
make
```

## Возможности

- выбор алгоритма из подключенных динамических библиотек;
- шифрование и дешифрование текста;
- шифрование и дешифрование файла по произвольному пути;
- бинарная обработка файлов любых учебных форматов: `.txt`, `.pdf`, `.jpg`, `.png`, `.zip`, `.mp4` и других;
- создание директорий для выходного файла при необходимости;
- генератор учебных ключей в каждом плагине через функцию `cipherGenerateKey`;
- ввод ключа пользователем;
- вывод промежуточных этапов;
- сохранение результатов в отдельные файлы;
- проверка неверного пункта меню, пустого ввода и отсутствующего файла.

## Работа с файлами разных форматов

Для шифрования файла выберите алгоритм, затем пункт `Зашифровать файл по пути`.
Программа читает исходный файл в бинарном режиме, поэтому расширение файла не важно.
Можно указать путь к текстовому документу, изображению, архиву или видеофайлу.
Путь можно вставлять в кавычках, с экранированными пробелами, через `~` или в виде `file://...`.

Если путь для результата оставить пустым, программа создаст файл рядом с исходным:

```text
video.mp4 -> video.mp4.encrypted
```

Для восстановления выберите `Дешифровать файл по пути`, укажите зашифрованный файл и путь для результата.
Если путь результата оставить пустым, будет создан файл с префиксом `decrypted_`.

```text
video.mp4.encrypted -> decrypted_video.mp4.encrypted
```

Для удобной проверки лучше явно указать имя результата с исходным расширением, например:

```text
video_restored.mp4
```

## Ввод длинного текста

Обычный однострочный текст можно вставлять сразу после приглашения.
Для длинного многострочного текста используйте режим:

```text
<<<
здесь может быть длинный текст
в несколько строк
<<<END>>>
```

Программа также очищает служебные маркеры bracketed paste, которые некоторые терминалы добавляют при большой вставке.

## API плагина

Каждая библиотека экспортирует функции:

- `cipherName`
- `cipherEncryptedFile`
- `cipherDecryptedFile`
- `cipherKeyHint`
- `cipherGenerateKey`
- `cipherEncrypt`
- `cipherDecrypt`
- `cipherPrintInfo`

Основная программа не подключает заголовки конкретных алгоритмов и не линкует их напрямую. Она загружает библиотеки из папки `plugins` через `dlopen`/`dlsym`.

## Генерация ключей

Пункт меню `Генератор ключей` вызывает функцию `cipherGenerateKey` выбранного динамического модуля.
Пустой ввод ключа при шифровании или дешифровании также вызывает этот генератор.

В учебной версии генерируются небольшие параметры:

- RSA: небольшие простые `p`, `q`, открытый показатель `e` и обратный `d`; в ключ пользователю выводится `n e d`;
- Рабин: простые `p` и `q`, подходящие для демонстрационного восстановления байтов через тетрады;
- Месси - Омура: простое `p`, показатель `e` и обратный показатель `d`;
- Шамир: простое `p` и пары взаимно обратных показателей `cA/dA`, `cB/dB`;
- ChaCha20: 256-битный ключ в hex, 96-битный nonce в hex и счетчик;
- А1Я33: параметры `a`, `b` и обратный элемент `inverseA` для русского алфавита из 33 букв.

Это демонстрационная учебная генерация для РГР. Она нужна для понимания алгоритмов и не является промышленной криптографией.

## Примечание

Реализация учебная: для асимметричных алгоритмов используются небольшие числа и типы `int`/`unsigned int`.
Скомпилированные файлы (`encryption_rgr`, `.o`, `.so`, `.dylib`, папка `plugins`) не должны храниться в репозитории и исключены через `.gitignore`.
